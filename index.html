What is syndatakit

A generator,
not a database.

Running syndatakit generate hmda --rows 1000000 creates one million brand new synthetic records. Nothing is stored. Everything generated on demand, every time.

Generator types

Four algorithms.
One interface.

Every generator inherits from BaseGenerator — same fit(), same sample(), regardless of algorithm.

Standard · No extras
Gaussian Copula
Fits marginal distributions per column, learns cross-column correlations via Cholesky decomposition. 96–98% fidelity on most tabular data. Auto-selected for cross-sectional datasets.
Filter by any column combination
Bayesian prior regularisation
Works with any CSV schema
Advanced
VAR / VECM+GARCH
Vector Autoregression for temporal datasets. VECM handles cointegration. GARCH(1,1) models volatility clustering — correlations that spike in crises. Auto-selected for fred_macro, bls.
Johansen cointegration test
ADF stationarity test per column

DP Gaussian Copula
Formal (ε,δ)-DP guarantee. Laplace mechanism on marginal statistics, Wishart mechanism on the correlation matrix. Budget tracked and reported. GDPR Article 89 compatible.
ε configurable — 0.1 to 10.0

Vine Copula
Pair copula construction — each column pair gets its own family (Clayton, Gumbel, Frank, Joe) chosen by AIC. Captures asymmetric tail dependence: correlations higher in crises than normal markets.

generate_credit_risk.py
from syndatakit.generators  import GaussianCopulaGenerator
from syndatakit.catalog     import load_seed
from syndatakit.calibration  import get_priors, apply_scenario
from syndatakit.fidelity    import fidelity_report

# Fit with Bayesian priors — stable even on small datasets
gen = GaussianCopulaGenerator(priors=get_priors("credit_risk"))
gen.fit(load_seed("credit_risk"))

# 100K high-risk records with filters
df = gen.sample(100_000, filters={
    "fico_band":   ["300-579", "580-669"],
    "dti_min":     45,
    "default_12m": 1,
})

# Apply credit crisis scenario
stressed = apply_scenario(df, "credit_crisis", intensity=0.9)

# Full fidelity report
report = fidelity_report(load_seed("credit_risk"), df)
print(f"Fidelity: {report['summary']['overall_fidelity']}%")

api_example.py
import requests, pandas as pd

# Start the server first: syndatakit serve --port 8080

# Generate equity returns under recession
r = requests.post("http://localhost:8080/generate", json={
    "dataset":   "equity_returns",
    "rows":      5000,
    "scenario":  "recession",
    "intensity": 0.9,
})
df = pd.DataFrame(r.json()["data"])

# Upload your own CSV and generate from it
r2 = requests.post("http://localhost:8080/generate/custom",
    files={"file": open("my_data.csv")},
    data={"rows": "100000"},
)

# Full privacy audit
r3 = requests.post("http://localhost:8080/audit",
    files={"real": open("real.csv"), "synthetic": open("syn.csv")},
    data={"attacks": "500"},
)
print(r3.json()["data"]["verdict"]["recommendation"])

Fidelity evaluation

Six dimensions.
One report.

Every generated dataset can be automatically evaluated across six independent metrics via fidelity_report(real, synthetic).

📊
Marginal
Kolmogorov-Smirnov for numeric columns. Total Variation Distance for categorical. Per-column scores 0–100%.
KS test · TVD
🔗
Joint
Spearman correlation matrix distance. Detects when the generator preserves marginals but breaks relationships.
Frobenius norm
📈
Temporal
ADF stationarity, Engle-Granger cointegration, Chow structural breaks, Granger causality direction.
ADF · VECM · Chow
📉
Stylized Facts
Fat tails (kurtosis), skewness sign, autocorrelation, ARCH volatility clustering. Critical for financial returns.
Kurtosis · ARCH
🤖
Downstream (TSTR)
Train on synthetic, test on real. Compare to training on real data. Ratio near 1.0 = perfect substitute.
TSTR / TRR
🔐
Privacy
Exact copy check, membership inference AUC, singling-out rate, linkability risk. Full verdict with recommendation.
MI · SO · LK

Privacy & compliance

Four tests.
One verdict.

Four independent attacks run automatically. Each returns a risk level. The overall verdict is the maximum risk across all tests, with a concrete recommendation.

Exact Copy Check
Zero tolerance
Hashes every synthetic row, checks against real data. Count must be 0. Always passes on properly fitted generators.
Membership Inference
AUC test
Shadow model attack. AUC near 0.5 = no advantage. AUC above 0.7 = memorisation risk detected.
Singling-Out
QI attack
Random quasi-identifier subset attack. Can an adversary uniquely identify one individual by matching on 2–4 attributes?
Linkability
NN attack
Nearest-neighbour cross-dataset attack. Can an adversary link a synthetic record back to a specific real individual? Baseline 0.5.
Differential Privacy
Formal (ε, δ)-DP guarantee via Laplace mechanism on marginals and Wishart mechanism on the correlation matrix. Budget consumed and reported per operation.
