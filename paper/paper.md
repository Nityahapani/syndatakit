# Summary

Empirical research in economics and finance often depends on datasets that are
restricted under data use agreements, delayed by multi-month review processes,
or impossible to share across institutions. `syndatakit` is an open-source
Python package that generates synthetic tabular data statistically calibrated
to real economic and financial datasets, enabling researchers to prototype
analyses, share reproducible workflows, and train models without accessing
restricted microdata.

The package implements a Gaussian copula generator with a library of
distribution-specific marginals (log-normal, truncated normal, beta, Pareto,
regime-switching mixture, zero-inflated), a VAR(1) generator for panel and
time series data, a differential privacy wrapper following the
$(\epsilon, \delta)$ framework of @dwork2006, and a Vine copula generator
[@joe1996] for asymmetric tail dependence. Eighteen built-in dataset profiles
are included, covering macroeconomic indicators, mortgage origination, bank
call reports, equity returns, corporate bonds, insurance, real estate, tax
statistics, and commodity markets. Each profile is calibrated against
published aggregate statistics from cited government sources. Generated
datasets contain zero real individuals and require no data use agreements.

Fidelity of generated data is measured against three dimensions: marginal
moment agreement, a two-sample Kolmogorov-Smirnov distributional test, and
Spearman rank correlation distance. The combined fidelity score across all 18
built-in profiles ranges from 81.7% to 95.0%. All validation results are
fully reproducible from `validation/fidelity_engine.py` in the repository.

# Statement of Need

A researcher studying mortgage lending discrimination needs HMDA loan
application data. A macroeconomist needs quarterly GDP and inflation series
for a forecasting exercise. A finance PhD student needs equity return panels
for a factor model. In each case, the real data is either confidential,
requires institutional affiliation and approval, or cannot be shared in a
replication package. The researcher either waits months for access, omits the
exercise entirely, or uses random numbers that bear no statistical resemblance
to the real data.

Existing synthetic data tools do not adequately address this problem for
econometric research. The Synthetic Data Vault [@patki2016synthetic] and
ydata-synthetic are designed for machine learning practitioners who need
additional training rows. They preserve marginal distributions and Pearson
correlations but do not model cointegration, GARCH volatility clustering
[@bollerslev1986], or the long-run equilibrium relationships that applied
econometrics depends on. Data generated by these tools routinely fails
standard time series tests - Augmented Dickey-Fuller [@dickey1979], Johansen
cointegration [@johansen1991] - and produces biased regression coefficients
in panel settings, which defeats the purpose for empirical researchers.

`syndatakit` addresses these gaps by selecting the statistically appropriate
generative model for each variable type and data structure. Inflation is
modelled as a regime-switching mixture to reflect the structural difference
between normal and high-inflation environments. Income variables use a
three-group log-normal mixture reflecting the empirical bracket distribution
from IRS Statistics of Income. Power-law variables such as firm revenue and
bank assets use a Pareto distribution with median-calibrated scale, scored
using robust estimators (median and IQR) because sample standard deviation is
unreliable when theoretical variance is infinite. The fidelity scorer uses
Spearman rank correlation rather than Pearson because the Gaussian copula
preserves rank structure by construction, and Pearson is systematically
attenuated for heavy-tailed distributions even when the copula is correctly
specified. A downstream OLS comparison test - in which the same regression
specification is run on real and synthetic data and coefficient vectors are
compared - provides validation at the level of econometric practice rather
than distributional summary statistics alone.

The package targets researchers in economics, finance, and adjacent fields
who need statistically credible synthetic data for teaching, prototyping,
privacy-preserving data sharing, and replication packages. It has been
designed for use alongside standard econometric workflows in Python, R (via
`reticulate`), and Stata (via a companion `.ado` plugin).
